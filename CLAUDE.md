# Раздел I. Техническое задание

## 1. Название проекта

- **Полное название:** Интернет-магазин цифровых игр «EZKERE»
- **Условное обозначение:** ezkere.ru
- **Домен:** ezkere.ru

## 2. Назначение системы

Веб-приложение предназначено для продажи цифровых копий компьютерных и консольных игр. Система предоставляет пользователям каталог игр с фильтрацией по жанрам и платформам, корзину покупок, личный кабинет с балансом и историей заказов. После оплаты пользователь получает уникальный токен заказа и контакт менеджера в Telegram для получения ключа активации.

## 3. Основные цели создания проекта

1. Создать удобную и визуально привлекательную витрину цифровых игр с адаптивным дизайном.
2. Реализовать полный цикл покупки: от просмотра каталога до оформления заказа и получения токена.
3. Обеспечить работу магазина на всех типах устройств — от мобильных телефонов до сверхшироких мониторов.
4. Предоставить систему учётных записей с балансом и историей покупок, синхронизируемую между устройствами через серверную базу данных (PocketBase).
5. Построить масштабируемую архитектуру, допускающую дальнейшее подключение реальных платёжных систем (ЮKassa, CloudPayments).

## 4. Задачи, которые решает сайт

| Задача | Описание |
|---|---|
| Просмотр каталога | Пользователь может просматривать 73 игры, фильтровать по 16 жанрам и 3 платформам, сортировать по цене, рейтингу, дате |
| Поиск игр | Мгновенный поиск по названию (RU/EN) и тегам с автодополнением |
| Детальная информация | Страница каждой игры содержит описание, системные требования, рейтинг Metacritic, ссылки на магазины |
| Корзина | Добавление/удаление товаров, подсчёт итоговой суммы с учётом скидок |
| Регистрация и авторизация | Создание аккаунта, вход по email+пароль, синхронизация данных между устройствами |
| Оформление заказа | Списание средств с баланса, генерация 10-значного токена, инструкция по получению товара |
| Личный кабинет | Просмотр баланса, истории покупок, пополнение баланса |
| Переключение темы | Тёмная (по умолчанию) и светлая тема оформления |

## 5. Целевая аудитория

- **Основная:** геймеры 16–35 лет, приобретающие цифровые копии игр для PC, PlayStation и Xbox.
- **Вторичная:** пользователи, ищущие выгодные скидки на популярные игры.
- **География:** Россия и СНГ (интерфейс на русском языке, цены в рублях).

## 6. Основные функции (действия пользователя)

1. **Просмотр главной страницы** — баннер с рекомендуемой игрой, лента трендовых игр, превью категорий.
2. **Навигация по каталогу** — фильтрация по жанру (16 жанров), платформе (PC/PS/Xbox), сортировка.
3. **Поиск** — ввод запроса в строку поиска, получение результатов с автодополнением (до 5 совпадений).
4. **Просмотр карточки игры** — описание, скриншоты, системные требования, оценка Metacritic, ссылки на Steam/PS Store/Xbox Store.
5. **Управление корзиной** — добавление, удаление товаров, просмотр итоговой суммы.
6. **Регистрация** — ввод имени пользователя, email, пароля.
7. **Авторизация** — вход по email и паролю.
8. **Пополнение баланса** — ввод суммы в личном кабинете.
9. **Оформление заказа** — выбор способа оплаты, подтверждение, получение токена.
10. **Переключение темы** — кнопка в шапке для смены тёмной/светлой темы.
11. **Боковое меню** — навигация по категориям на мобильных устройствах.

## 7. Требования к безопасности

| Требование | Реализация |
|---|---|
| Хеширование паролей | Пароли не хранятся в открытом виде. PocketBase хеширует пароли с использованием bcrypt |
| Аутентификация по токенам | JWT-токены для сессий пользователей (PocketBase Auth) |
| Криптографическая генерация токенов заказов | `crypto.getRandomValues()` для генерации 10-значных токенов |
| Защита API | PocketBase обеспечивает правила доступа к коллекциям (API Rules) |
| HTTPS | SSL-сертификат через Let's Encrypt (Certbot) |
| Защита от XSS | React автоматически экранирует все выводимые значения |
| CORS | Настройка допустимых доменов в PocketBase |

## 8. Требования к производительности

| Параметр | Значение |
|---|---|
| Время первой загрузки (LCP) | < 2 секунды |
| Размер бандла (gzip) | ~132 КБ JS + ~6 КБ CSS |
| Поддерживаемые разрешения | 320px — 3840px (от мобильных до 4K) |
| Одновременные пользователи | До 1000 (ограничение PocketBase на одном сервере) |
| Кэширование статики | Nginx отдаёт файлы из `/dist` с заголовками кэширования |
| Оптимизация изображений | Обложки загружаются с CDN Steam (lazy loading) |

## 10. Причины выбора технологий

| Технология | Причина выбора |
|---|---|
| **React** | Самый распространённый UI-фреймворк с крупнейшей экосистемой. Компонентный подход позволяет переиспользовать элементы (GameCard, SearchBar). Virtual DOM обеспечивает быстрый рендеринг каталога из 73 игр |
| **Vite** | В 10-20 раз быстрее Webpack. Мгновенный HMR при разработке (< 50ms). Оптимизированная продакшн-сборка с tree-shaking и code-splitting |
| **React Router** | Стандарт маршрутизации для React-приложений. Поддержка вложенных маршрутов, параметров URL, query-строк |
| **CSS Modules** | Автоматическая изоляция стилей — классы не конфликтуют между компонентами. Нет необходимости в CSS-in-JS библиотеках |
| **PocketBase** | Один исполняемый файл заменяет бэкенд + базу данных + авторизацию. Встроенный admin UI. Не требует Docker, Node.js или отдельного сервера БД. Идеален для MVP |
| **Nginx** | Высокопроизводительный веб-сервер для статики. Встроенное проксирование API. Интеграция с Let's Encrypt |

## 16. Ключевые компоненты

### ThemeContext — переключение тем

Реализует переключение между тёмной и светлой темой. Применяет атрибут `data-theme` к элементу `<html>`, что автоматически переключает все CSS-переменные. Выбранная тема сохраняется в localStorage.

### AuthContext — авторизация через PocketBase

Управляет всем циклом пользовательской сессии: регистрация, вход, выход, пополнение баланса, оформление покупок. Взаимодействует с PocketBase через JS SDK. Функция `normalize()` приводит записи PocketBase к формату, ожидаемому компонентами приложения.

### CartContext — корзина с синхронизацией

Гибридное хранение: для авторизованных пользователей — PocketBase (синхронизация между устройствами), для гостей — localStorage (локальная корзина). Автоматическое переключение при логине/логауте.

### SearchBar — поиск с автодополнением

Поиск по трём полям (title, titleRu, tags) с дебаунсом 300мс. Выпадающий список до 5 совпадений с обложками и ценами. На мобильных устройствах (< 768px) скрыт и открывается по кнопке в шапке.

### GameCard — карточка игры

Отображает обложку, название, цену (с учётом скидки), рейтинг Metacritic с тултипом. Цвет рейтинга: зелёный (>= 75), жёлтый (>= 50), красный (< 50). Для анонсированных игр (score = 0) рейтинг не отображается.

### generateToken — криптографический генератор

Использует `crypto.getRandomValues()` для генерации случайного 10-значного токена из букв и цифр. Применяется при оформлении заказа для создания уникального идентификатора, который пользователь передаёт менеджеру.

## 17. Решение технических задач

### Задача 1: Синхронизация данных между устройствами

**Проблема:** Изначально все данные хранились в localStorage — аккаунт, корзина, история покупок. При входе с другого устройства данные терялись.

**Решение:** Подключение PocketBase в качестве серверной базы данных. AuthContext и CartContext переписаны для работы с PocketBase REST API через JS SDK. JWT-токен сессии хранится в localStorage автоматически через `pb.authStore`.

### Задача 2: Адаптивный поиск на мобильных устройствах

**Проблема:** Строка поиска (SearchBar) занимает много места и скрыта на экранах < 768px, но пользователи мобильных устройств тоже должны иметь возможность искать.

**Решение:** В Header добавлена кнопка-лупа (`mobileSearchBtn`), видимая только на мобильных. При нажатии открывается полноширинная строка поиска (`.mobileSearchRow`) под шапкой сайта. При выборе результата или отправке формы строка автоматически закрывается.

### Задача 3: Безопасная генерация токенов заказов

**Проблема:** Токен заказа должен быть непредсказуемым и уникальным, чтобы его нельзя было подобрать перебором.

**Решение:** Использование Web Crypto API (`crypto.getRandomValues()`) вместо `Math.random()`. Алфавит из 62 символов (A-Z, a-z, 0-9) и длина 10 символов дают 62^10 = 839 квадриллионов возможных комбинаций.

### Задача 4: Расчёт цен со скидками

**Проблема:** Необходимо корректно отображать и рассчитывать цены с учётом процентных скидок и форматировать их в рублях.

**Решение:** Утилиты `formatPrice()` и `calcDiscount()` в `src/utils/formatPrice.js`. `calcDiscount` вычисляет финальную цену с округлением, `formatPrice` форматирует через `Intl.NumberFormat('ru-RU')` с символом рубля.

### Задача 5: Переключение тем без перезагрузки

**Проблема:** При переключении темы все элементы интерфейса должны мгновенно менять цвета без перерисовки или мигания.

**Решение:** Все цвета вынесены в CSS-переменные. `ThemeContext` переключает атрибут `data-theme` на `<html>`, что мгновенно активирует соответствующий блок CSS-переменных. Выбранная тема сохраняется в localStorage и восстанавливается при загрузке.
